WCAG 2.4.3 Focus Order Test

Test Type: Positive tabindex values (WCAG Failure)

Expected Behavior

This page should FAIL WCAG 2.4.3 because it uses positive tabindex values that create confusing focus order.

Expected Tab Order:

  1. Button 3 (tabindex="1")
  2. Button 2 (tabindex="2")
  3. Button 5 (tabindex="3")
  4. Button 1 (tabindex="0" or natural order)
  5. Button 4 (tabindex="0" or natural order)
  6. Input Field
  7. Submit Button

Why This Fails WCAG 2.4.3

  • Positive tabindex values (1, 2, 3) override the natural DOM order
  • Focus jumps around unpredictably (Button 3 → Button 2 → Button 5 → Button 1)
  • Creates fragile focus order that breaks when content is added/removed
  • Violates user expectation of top-to-bottom, left-to-right navigation

Correct Implementation

To fix: Remove all positive tabindex values and rely on natural DOM order, or use tabindex="0" for custom widgets.
